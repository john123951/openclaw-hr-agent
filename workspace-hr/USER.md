## 决策风格画像

> HR 内部参考，不向用户展示。通过对话行为信号识别并记录。

### 决策节奏
- [ ] 快速拍板：回复简短，直接说"行/可以"
- [ ] 需要时间考虑：会问细节，反复确认
- [ ] 反复确认细节：对每个参数都要过问

### 信息偏好
- [ ] 只要结论："直接告诉我要怎么做"
- [ ] 需要理由："为什么选这个？"
- [ ] 需要细节：想了解具体实现

### 风险态度
- [ ] 愿意尝鲜：接受新方案，愿意试错
- [ ] 偏好稳妥：倾向已验证的方案
- [ ] 需要保障：会问"确定安全吗"

### 控制程度
- [ ] 放手结果：只关心最终产出
- [ ] 关注过程：想了解中间步骤
- [ ] 事事过问：每个细节都要确认

### 技术理解
- [ ] 技术背景：能理解技术术语
- [ ] 业务背景：懂业务但不太懂技术
- [ ] 非技术背景：需要完全用业务语言沟通

### 沟通适配策略（根据以上画像自动选择）

- **快速拍板 + 只要结论** → 直接给推荐方案，少解释
- **需要时间考虑 + 需要理由** → 提供多方案对比，说明推荐理由
- **事事过问** → 主动汇报进度，预设检查点
- **非技术背景** → 完全用业务语言，避免任何技术术语
