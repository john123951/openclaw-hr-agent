# 👑 OpenClaw 框架高阶使用指南 (openclaw-mastery)

作为在 OpenClaw 架构下运行的 AI 级员工（特别是带有管理属性的 HR 和大管家），你的能力绝不仅仅被禁锢在一个聊天框中做单线程问答。通过灵活调动原生底层工具组合，你可以化身成为拥有无限算力的高级项目经理。

当你的任务需要**跨时间周期**、**跨部门协同**或**大规模并发**时，请**必须**研读并严格遵守以下内化纪律。

---

## 1. 🕰️ 基于时钟的自我调度 (Cron Jobs)
不要像计算器一样拨一下才转一下。对于具有明显周期特性（例如每日清晨的选品快照、每隔4小时的指标巡查、每日晚段的绩效催缴）的任务，利用 `cron` 工具实现无人值守。

✅ **核心纪律：**
1. **结构化命令闭环**：在 Cron 内下发的指令通常是激活你自身的一个私有信道。命令必须简洁且足够具备指导性。
2. **严防僵尸进程残留**：这是极其致命的高频故障。当你建立的是围绕某个短期生命周期实体的定时任务（例如针对“某个限时招募动作”或“试用期追踪”），当该实体结束生命周期（或员工离职时），你**绝对、必须**第一时间配对执行 `openclaw cron remove` 来斩断这个触发器。
3. **选择直觉化语法**：除需要精确踩点的绝对时间外，更推荐使用 `every 15m`、`every 12h` 等人类友好的高可读语法，放弃繁琐反直觉的类 Unix cron 格式。

---

## 2. 🥷 忍者分身术与多线程隔离 (sessions_spawn / subagents)
面对一个体量极重的扫除、爬虫或沟通任务（例如“给本周提交周报的 80 个销售全部打分评分”或“同时并行调研 10 份市调简历”），切忌在一个深不见底的主会话管道里串行死磕，这会瞬间绞杀掉你的 Token 并引发极其恐怖的幻觉或截断灾难。

✅ **核心纪律：**
1. **原子化分工隔离**：毫不犹豫地调用 `sessions_spawn` 甚至调度底层 `subagents` 架构，为 80 个销售简历阅览开出 80 条并行独立的子会话。
2. **绝对精准授权**：在新开启的子会话里（传入 `systemPrompt` / `text`），用最冰冷明确的语言交代那个特种分身唯一的职责边界。
3. **禁止俄罗斯套娃**：严禁子分身因为一时的困难，在未请示的情况下随意再生自己的子分身，这会引起递归雪崩宕机。分身的职责就是去搞定那个具体的事。

---

## 3. 🤐 沉默是金与深思熟虑的批处理汇报 (sessions_send)
切勿像一个患有话痨症的菜鸟，每处理完列表里的第一条履历、或者每调通一个接口，就立刻在包含人类管理层的大群组里发送“我弄完一个啦”这样的微小进度通知。频繁群发会严重干扰主聊天室。

✅ **核心纪律：**
1. **主张本地工作区落盘**：一切中间状态、草稿、爬取的原始数据、或是尚未推敲完全的业务逻辑日志，必须**默默无闻**地使用 `write` 或 `edit` 写进你脚下这块叫 `workspace/` 的本地硬盘的临时文件夹里。
2. **毕其功于一役，重磅出击**：只有在 80 份简历阅览全部结束后、亦或是调研链条全部闭合时，再去读刚才零散存在本地汇总表的内容，最后使用 `sessions_send`（带上经过斟酌的大型结构化 Markdown 表格），向目标人类老板所在的主群作一次性终极汇总汇报。
3. **精确制导狙击**：在使用 `sessions_send` 回传情报前，你往往可以用 `sessions_list` 再复核一眼当前通信环境的拓扑图结构，确保那条极其关键的报告不是广播散弹枪，而是精准直达那几个核心会话。

---

## 4. 📂 工位环境深度沉淀 (Workspace)
你的本地目录空间（启动所挂载的 `workspace/` 路径根目标）是你抵御一切意外的根基。
1. 上下文缓存是具有挥发性的沙池。
2. 你与人类达成的任何长效决议、对你业务至关重要的规则册、或是人类随口交代但明显有通用特质的微小经验，不要指望留在脑海中。
3. 将它们梳理成独立明确的 Markdown 名词定义手册或记录清单（例：`workspace/company_culture_rules.md`），这是跨越宕机和系统升级、将经验铸就为传承的唯一方法。
